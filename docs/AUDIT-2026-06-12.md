# Full Codebase Audit + QA — 2026-06-12

Four parallel review agents (storefront, admin, mobile, database/security) plus
manual verification. Below: what was FIXED this pass, and what is DEFERRED with the
recommended fix. Shipped in commit `d8e26e0` (web auto-deploys; mobile via EAS
preview update) + migration 0076.

## FIXED

### Critical
- **Price-region exploit (storefront).** `api/checkout` trusted the browser region
  cookie, so a shopper abroad could set it to ZA and pay the cheaper Rand product
  prices on an international order. Now the price region is locked to the shipping
  country (`regionForCountry(shipping.country)`), as region.ts always promised.
- **Delete-account could silently fail (mobile).** `settings.tsx` ignored the
  `delete_my_account` RPC error and signed the user out regardless. Now it verifies
  success before sign-out and surfaces failure. (CLAUDE.md: data-delete is sacred.)

### High
- **Chat + DMs loaded the OLDEST 100 messages.** `lib/guild.ts` + `lib/social.ts`
  ordered ascending; once a channel passed 100 messages, recent ones were invisible.
  Now fetch newest-100 descending, reversed for display.
- **CRON endpoints were open.** `CRON_SECRET` was unset, so `if (secret && ...)`
  skipped auth entirely: anyone could trigger the daily-reminder (mass email) and
  feedback-digest routes. Set `CRON_SECRET` in Vercel (prod/preview/dev) and both
  routes now fail closed.
- **Broadcast reachable by moderators.** `sendBroadcast`/`sendTestEmail` message
  every member but had no caller check. Now admin-only (`isCallerAdmin`); moderator
  access to the broadcast page removed (middleware + nav).

### Medium
- **DM thread deadlock (mobile).** `dm/[id].tsx` used `null` for both "loading key"
  and "no key", so a thread with a keyless partner never loaded. Now `undefined` =
  loading, `null` = no key.
- **Feed had no ORDER BY before limit(50) (mobile).** Past 50 posts the feed could
  drop recent ones. Now ordered newest-first server-side + filters the app target.
- **Two liturgical engines disagreed on Orthodox fasts by 13 days.** `liturgical.ts`
  used Julian dates; `calendar.ts` used revised-Julian. Unified `liturgical.ts` to
  the revised-Julian civil dates (Nativity Nov 15-Dec 24, Dormition Aug 1-14) and
  added the Apostles' Fast. Regression suites pass (qa-office 68, qa-calendar 60).
- **Discount usage race (storefront).** Read-then-write of `used_count` could lose
  updates. Now an atomic `bump_discount_use` RPC (migration 0076).
- **Donations hardcoded ZAR.** Now use the visitor's region currency.
- **Dead code + duplication (admin).** Removed unused `updatePost`; deduped
  `anthropicConfig` and the signed-upload-ticket logic into `lib/anthropic-config.ts`
  and `lib/storage-upload.ts` (shared by Content Studio + Ad Studio).

### Low / hygiene (migration 0076)
- `design_files` given an explicit admin RLS policy (was deny-all by default).
- `day_history(status, day)` index for the daily-reminder cron.
- Review media URLs must be http(s) (blocks javascript:/data:).

## DEFERRED (with recommended fix)

- **`profiles.email` is readable by guild-mates (MEDIUM, privacy).** The
  `profiles_select_guild` RLS policy grants full-row reads, so a guild member could
  craft a query to harvest co-members' emails. The app never requests it in normal
  use. Not fixed now because the only safe fix (column-level REVOKE on email) would
  break `useProfile`'s `select("*")` (Profile has an open index signature). Recommended
  pre-launch fix: move email to a private own-row-only table (or switch `useProfile`
  to explicit columns + `REVOKE SELECT(email) ... FROM authenticated, anon`; service
  role/admin/cron keep access). Do before public launch.
- **No rate limiting on public POST routes (LOW).** /api/checkout, /donate, /review,
  /returns, /subscribe, /track. Plan (already noted in PERF-NOTES): Upstash rate limit.
- **No CSP headers (LOW).** Add a Content-Security-Policy in middleware/next.config.
- **Payment redirect not validated (LOW, not yet exploitable).** Providers are stubbed
  (return null). When a real gateway is wired, whitelist the redirect host + add
  webhook signature verification.
- **Missing audit logs on some admin mutations (LOW).** orders/discounts/content/
  reviews/donations/giveaway-delete don't call `logAudit`. Add for completeness.
- **Mobile non-atomic writes (LOW).** `toggleQuest` and the workout-complete path do
  two writes with errors swallowed; can diverge from server on failure. Wrap in an
  RPC or reconcile on next load.
- **Tabata timer drifts/pauses in background (LOW).** setInterval tick-counting;
  anchor to Date.now() deltas.
- **Stale XP bar / no day rollover while app stays open (LOW).** `useProfile` fetches
  once; the XP bar doesn't refresh on quest completion and quests don't roll to a new
  day without a restart. Add a focus refresh + a day-change check.
- **Mobile dead code (LOW).** `components/Card.tsx`, `components/Pillar.tsx`,
  `exploreGuilds`/`ExploreGuild` in lib/guild.ts are unused.
- **Chat render perf (LOW).** guild/DM screens map all messages in a ScrollView;
  every keystroke re-renders the list. Move to an inverted FlatList like feed.tsx.

## Verified clean

recompute, inventory, email escaping, discount math rounding. DB/RLS: integrations
(admin-only, secrets never client-exposed), push_tokens (user-scoped), orders
(user-scoped), posts scheduling policy, broadcasts (admin-only), ad_* (admin-only),
no NEXT_PUBLIC secret leakage, supabaseAdmin server-only. Admin: grant/revoke admin,
settings, deploy/social-status API routes all properly authorized. Mobile: game math
via @tabor/shared (no drift), UTC date handling, realtime channel cleanup, SecureStore
key handling, push registration.
